# Migration Notes

## What changed

The desktop JavaFX client was replaced with a browser SPA. The backend remains the core REST API and now owns all security, authorization, ownership, business validation, and clean error handling.

## Desktop-to-web mapping

| JavaFX controller | Web route/workflow |
|---|---|
| `StudentsAdminController` | `/admin/students`, `/admin/students/new`, `/admin/students/:id`, `/admin/students/:id/indexes` |
| `StudentsSearchController` | `/admin/students` search/pagination |
| `StudentsByIndexController` | Admin quick lookup through secured student endpoints |
| `StudentsBySrednjaController` | Admin high-school search extension point |
| `StudentProfileController` | `/student/profile`, `/student/dashboard`, admin detail screens |
| `NastavniciController` | `/admin/professors` and professor subject views |
| `ProgramsController` | `/admin/programs`, `/admin/subjects` |
| `ExamsController` | `/admin/exams`, `/professor/exams`, `/professor/exams/:id/results` |
| `ReportsController` | `/admin/reports`; report PDFs should be generated by the backend |

## Not ported directly

- JavaFX controllers and FXML layouts.
- Desktop-only navigation and dialog code.
- Desktop RestTemplate clients.

## Kept/reused

- Existing Spring Boot backend modules.
- Existing domain entities and repositories where usable.
- Existing DTO package style.
- JasperReports approach as a backend-only extension if PDF reports are required.

## Important production notes

- Production schema is migration-managed. Do not use `ddl-auto=update` in production.
- Create real `UserAccount` rows with BCrypt hashes before deployment.
- Confirm tuition rules and registration-deadline rules with business owners.
- Validate migration SQL against the exact production MySQL version before running it on a live database.
